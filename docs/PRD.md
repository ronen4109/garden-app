# PRD — אפליקציית גינה ביתית

## 1. רקע ומטרה

משפחה בבת חפר עם 6 עצי פרי בחצר (רימון, אפרסק, מנגו, קלמנטינה, תפוז, לימון). אין ידע מוקדם בגינון. צריך כלי פשוט שיגיד מה לעשות, מתי, ולכל עץ — ושיאפשר לעקוב לאורך השנה.

**מטרה:** אפליקציית ווב מינימליסטית שעונה על שלוש שאלות:
1. מה צריך לעשות היום/השבוע?
2. מתי טופל העץ הזה לאחרונה?
3. מה ההיסטוריה של העץ?

## 2. עקרונות יסוד

- **בלי הגזמות.** כל פיצ'ר שלא משרת ישירות את 3 השאלות לעיל — לא נכנס.
- **בלי משתמשים, בלי login, בלי סיסמה.** URL פתוח. נכנסים ומשתמשים.
- **המכשיר הראשי הוא הטלפון.** לא דסקטופ.
- **שפה: עברית, RTL.**
- **תכנון לכל השנה.** האפליקציה יודעת מה צריך לעשות בכל חודש לכל עץ.

## 3. משתמשים

המשתמשים הם רונן ובת/בן הזוג. אין הפרדה ביניהם באפליקציה — מצב משותף. אם מישהו רוצה לציין מי עשה משהו, יכתוב את זה בהערה החופשית.

## 4. Stack טכני

- **Frontend:** Next.js 16 (App Router)
- **Hosting:** Vercel
- **Database:** Neon Postgres (דרך Vercel Marketplace) — יש שכבה חינמית מספיקה
- **ORM:** Drizzle (קל, TypeScript-first)
- **UI:** shadcn/ui + Tailwind, RTL
- **שליחת התראות שבועיות:** Vercel Cron + Resend (מייל) — שלב 2

## 5. פיצ'רים — MVP

### 5.1 דף הבית — "מה היום"

- ברירת מחדל של הכניסה לאפליקציה
- מציג עד 5 משימות פתוחות, ממוינות לפי דחיפות (איחור > היום > השבוע)
- כל משימה: שם עץ + פעולה + תאריך יעד + כפתור "סיימתי"
- כפתור גדול "סיימתי" → סוגר את המשימה ולוג את התאריך
- ניווט תחתון: בית | עצים | לוח שנה

### 5.2 רשימת עצים

- גריד 2×3, תמונה + שם של כל עץ
- לחיצה → דף עץ

### 5.3 דף עץ

תוכן:
- תמונה גדולה + שם בעברית
- **משימות פתוחות לעץ הזה** (כל מה שצריך לעשות בקרוב)
- **תכנית הטיפול השנתית** (collapsible) — מה צריך לעשות לעץ הזה בכל חודש
- **היסטוריה** — 20 פעולות אחרונות שבוצעו (תאריך + פעולה + הערה)
- **תצפיות** — כל ההערות החופשיות שהוקלדו לעץ הזה
- כפתור "הוסף תצפית" — טקסט חופשי + תמונה אופציונלית

### 5.4 לוח שנה

- תצוגה חודשית
- כל יום שיש בו משימות מסומן עם נקודה
- לחיצה על יום → רואים את המשימות של היום

### 5.5 סימון משימה כבוצעה

- כפתור "סיימתי" בכל משימה
- אופציונלי: שדה הערה ("הוספתי גם קומפוסט", "ראיתי שיש כנימות")
- תאריך נשמר אוטומטית
- המשימה הבאה מאותו סוג מתוזמנת אוטומטית לפי הקבוע (למשל: השקיה כל 3 ימים)

### 5.6 הוספת תצפית

- בחירת עץ
- טקסט חופשי
- תמונה אופציונלית (העלאה מהגלריה / מצלמה)
- נשמר בהיסטוריה של העץ

## 7. מודל נתונים

```
trees
  id              uuid PK
  name            text          ("רימון", "אפרסק", ...)
  species         text          ("Punica granatum", ...)
  photo_url       text
  planted_date    date          (אופציונלי)
  notes           text          (אופציונלי)
  created_at      timestamp

care_plan_items   (תכנית הטיפול הקבועה — נטענת חד-פעמית מ-seed)
  id              uuid PK
  tree_id         uuid FK → trees
  action          text          ("השקיה", "דישון", "גיזום", "ריסוס", "חיפוי")
  description     text          ("דשן איטי-שחרור, חופן סביב הגזע")
  month           int           (1-12) או null
  frequency_days  int           (null אם פעם בשנה)
  season          text          ("אביב", "קיץ", "סתיו", "חורף") — אופציונלי

task_logs         (ביצוע בפועל)
  id              uuid PK
  tree_id         uuid FK → trees
  care_plan_id    uuid FK → care_plan_items (null אם ad-hoc)
  action          text
  done_at         timestamp
  note            text          (אופציונלי)

observations      (הערות/תצפיות חופשיות)
  id              uuid PK
  tree_id         uuid FK → trees
  text            text
  photo_url       text          (אופציונלי)
  created_at      timestamp
```

## 8. תכנית הטיפול (seed data)

בעת ההתקנה הראשונית, ה-DB ייטען עם תכנית הטיפול לכל 6 העצים — בערך 6-10 משימות לעץ בשנה. בסך הכל ~50 פריטים בטבלת `care_plan_items`.

דוגמה לרימון:
- מאי: דשן איטי-שחרור (פעם בעונה)
- יוני-אוגוסט: בדיקת חיפוי
- נובמבר: קומפוסט סביב הגזע
- דצמבר-ינואר: גיזום חורף
- ינואר: סיוד גזע

תכנית מפורטת תיכתב בקובץ נפרד `care-plans.md` ותומר ל-seed.

## 9. דרישות לא-פונקציונליות

- **ביצועים:** טעינה ראשונית < 2 שניות בנייד
- **נגישות בסיסית:** טקסטים בגדלים סבירים, קונטרסט תקין
- **RTL מלא**
- **תמיכה ב-PWA:** ניתן להוסיף למסך הבית של הטלפון ולפתוח כמו אפליקציה
- **תמונות:** Vercel Blob להעלאה. דחיסה אוטומטית לפני שמירה
